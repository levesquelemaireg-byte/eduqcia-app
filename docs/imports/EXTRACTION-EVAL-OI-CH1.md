### Vérification code wizard — option « 1 document à 2 perspectives »

Constat principal : ce n'est pas un comportement générique basé uniquement sur `nb_documents: 2`.

Logique de branchement exacte :

1. Choix utilisateur du mode documentaire (groupé vs séparé)

- Composants :
  - `components/tache/wizard/bloc3/templates/Bloc3TemplateStructure.tsx` (OI3.3 / OI3.4)
  - `components/tache/wizard/bloc3/templates/Bloc3TemplatePur.tsx` (OI3.5 et OI6.3)
- UI : `RadioCardGroup` avec options « Un seul document (...) » vs « Documents distincts ».
- Action déclenchée : `SET_PERSPECTIVES_MODE_WITH_MIGRATION`.

2. Résolution des composants Bloc 4 selon le comportement

- Fichier pivot : `components/tache/wizard/wizardBlocResolver.tsx`.
- Source de vérité des comportements : `lib/tache/wizard-bloc-config.ts`.
- Mapping actif :
  - OI3.3 / OI3.4 / OI3.5 -> `bloc4.type = "perspectives"` -> `Bloc4Perspectives`
  - OI6.3 -> `bloc4.type = "moments"` -> `Bloc4Moments`
  - OI6.1 (et OI6.2) -> `bloc4.type = "standard"` -> `Bloc4DocumentsHistoriques`

3. Branching runtime à l'étape documents

- `Bloc4Perspectives.tsx` et `Bloc4Moments.tsx` :
  - `mode === "separe"` -> 2 panneaux `DocumentSlotPanel` (documents distincts)
  - sinon (`mode === "groupe"`) -> 1 encadré « Document A » + 2 sous-sections (perspectives ou états)
- Migration des données groupé <-> séparé : `lib/tache/reducer-perspectives.ts`.
- Validation « Suivant » + publication :
  - `lib/tache/wizard-step-next-gate.ts`
  - `lib/tache/publish-tache-payload.ts`

Conclusion technique : l'option « 1 document à 2 perspectives » est disponible uniquement pour les comportements explicitement câblés en `bloc4.type = "perspectives"` ou `"moments"`, pas pour tous les comportements avec `nb_documents = 2`.

### Points bloquants identifiés

1. OI5 non disponible en création

- Les questions Q3 et Q11 relèvent de "Mettre en relation des faits".
- Dans `public/data/oi.json`, OI5 est `coming_soon`.
- Les comportements associés ne sont pas sélectionnables par le wizard (filtrage de sélection des comportements non actifs / sans `nb_documents`).

2. Comportement OI4.4 non disponible

- Q4 correspond au format "facteur explicatif + conséquence" avec réponse en numéros de documents.
- Dans `public/data/oi.json`, le comportement 4.4 est `coming_soon` et non sélectionnable.
- Le parcours non rédactionnel `causes-consequences` n'est pas implémenté côté composants wizard actifs.

3. L'option « 1 document à 2 perspectives » n'est pas universelle

- Q7 (OI3.4) : mode groupé disponible, permet bien une consigne à partir d'un seul document.
- Q10 (OI6.3) : mode groupé disponible, permet bien une consigne à partir d'un seul document.
- Q14 (OI6.1) : `nb_documents = 2` dans `oi.json` mais bloc 4 en mode `standard` uniquement, sans bascule groupé/séparé.
